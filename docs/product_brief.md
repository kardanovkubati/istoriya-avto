# История Авто: product brief

Дата: 2026-05-15
Статус: рабочее каноническое описание продукта для проектирования frontend, roadmap и коммуникации внутри команды.

## Что делает продукт

**История Авто** помогает людям, которые ищут автомобиль, получить сводную историю машины перед решением о покупке.

Пользователь может:

- найти автомобиль по VIN, госномеру или ссылке на объявление;
- увидеть безопасное превью найденных кандидатов;
- открыть один сводный отчет по VIN;
- загрузить свой отчет по любому автомобилю;
- получить балл за полезный загруженный отчет;
- потратить балл или подписочный лимит на открытие другого сводного отчета;
- сохранить доступ к открытому VIN навсегда, включая будущие обновления;
- поделиться сводным отчетом по временной share-ссылке;
- выгрузить отчет в PDF в оформлении Истории Авто.

## Главная идея

У людей, которые ищут автомобиль, часто уже есть отчеты. История Авто превращает эти разрозненные отчеты и объявления в нормализованные факты и строит собственный сводный отчет по автомобилю.

Полезная загрузка пополняет базу и дает пользователю балл. Балл можно использовать для открытия другого отчета. Поэтому продукт строится вокруг обмена ценностью: пользователь приносит полезные данные и получает доступ к данным, которые помогают проверить другую машину.

При этом публичное позиционирование не должно звучать как "обмен отчетами сторонних сервисов". Наружная формулировка: **сервис проверки истории автомобиля**.

## Что является нашим отчетом

В продукте есть два разных понятия:

- **Исходный отчет** - PDF или authless-ссылка, которую загрузил пользователь. Исходные отчеты хранятся закрыто и не показываются другим пользователям.
- **Сводный отчет Истории Авто** - наш собственный отчет по VIN, собранный из нормализованных фактов, загруженных отчетов, объявлений и внутренних наблюдений.

Пользовательский интерфейс показывает именно сводный отчет Истории Авто, а не копию исходного отчета.

## Базовые пользовательские сценарии

### Поиск автомобиля

Пользователь вводит VIN, госномер или ссылку на объявление. Система определяет тип запроса и показывает найденные кандидаты в preview-режиме.

Preview помогает выбрать нужный автомобиль, но не раскрывает ценные блоки полного отчета. До открытия не показываются ДТП, ограничения, залоги, розыск, владельцы, история пробегов, расчеты ремонта, юридические статусы и другие закрытые данные.

### Загрузка отчета

Пользователь может загрузить текстовый PDF-отчет или authless-ссылку на отчет. Отчет не обязан относиться к автомобилю, который пользователь хочет открыть.

Если отчет валиден, полезен и проходит правила начисления, пользователь получает 1 балл. Если отчет подозрительный, частичный или не проходит автоматические условия, он может быть принят в базу без балла или отправлен на ручную проверку.

### Открытие сводного отчета

Открытие full report работает по access service:

- если VIN уже открыт пользователю, повторный просмотр бесплатный;
- если есть активная подписка с доступным лимитом новых отчетов, сначала тратится лимит;
- если лимита нет, но есть баллы, списывается 1 балл;
- если нет лимита и баллов, пользователь должен загрузить полезный отчет или выбрать тариф.

Один балл или один подписочный лимит открывает сводный отчет по VIN навсегда для этого пользователя, включая будущие обновления.

### Share и PDF

Пользователь с доступом к отчету может создать share-ссылку на 7 дней. Share view доступен без авторизации, но не дает PDF download и не позволяет пересоздавать share-ссылки.

PDF export доступен только владельцу доступа и должен быть оформлен как отчет Истории Авто, без копирования сторонних отчетов.

## Продуктовые принципы

- Не показывать пользователям оригинальные отчеты сторонних сервисов.
- Не копировать стиль, тексты, иконки, визуальные шкалы, фирменную верстку и формулировки сторонних сервисов.
- Строить собственный отчет на базе нормализованных фактов.
- Не показывать названия сторонних сервисов как источники конкретных фактов в пользовательском отчете.
- Не хранить и не показывать персональные данные людей в пользовательском отчете.
- Не раскрывать full report без доступа.
- Не списывать балл без явного действия пользователя на открытие отчета.
- Не возвращать балл, если пользователь сам выбрал не тот автомобиль для открытия.
- Сохранять доступ к открытому VIN навсегда.
- Баллы не сгорают; подписочные лимиты сгорают по правилам тарифа.

## Для кого продукт

Целевая аудитория - люди, которые ищут автомобиль.

Не нужно искусственно делить первую версию на "главного" и "неглавного" покупателя. Важен контекст: человек проверяет конкретную машину, сравнивает варианты, уже может иметь сторонние отчеты и хочет получить больше информации без лишней сложности.

## Тон продукта

История Авто должна ощущаться как спокойный, серьезный и полезный инструмент проверки машины, а не как маркетплейс отчетов и не как витрина "купить проверку".

Тон интерфейса:

- ясный;
- уверенный;
- без давления;
- без псевдоэкспертного шума;
- с аккуратными предупреждениями там, где данные неполные, старые или конфликтуют.

## Что важно для frontend-дизайна

Frontend должен вести пользователя через два равноправных действия:

- проверить автомобиль;
- загрузить отчет и получить балл, если отчет полезен.

При этом полный отчет должен выглядеть как собственный качественный документ Истории Авто: понятный, структурированный, пригодный для внимательного чтения, share и PDF.

Дизайн не должен начинаться с идеи "купить отчет". Покупка тарифа - это один из способов получить доступ, но не смысл продукта.
